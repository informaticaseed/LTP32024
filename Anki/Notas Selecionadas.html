O que é uma mensagem na POO? "
Em Programação Orientada a Objetos (POO), uma mensagem é uma solicitação que um objeto faz para outro objeto, pedindo que ele execute uma ação ou forneça uma informação.
Como funciona:
  • Objeto Emissor: O objeto que envia a mensagem.
  • Objeto Receptor: O objeto que recebe a mensagem.
  • Método: A ação ou função específica a ser executada pelo objeto receptor.
Exemplo:
Imagine que você tem uma classe Cachorro com um método latir().
  • Objeto Emissor: Uma pessoa (que pode ser representada por um objeto Pessoa).
  • Objeto Receptor: Um cachorro (representado por um objeto Cachorro).
  • Mensagem: ""Latir!""
  • Método: latir()
A pessoa envia a mensagem ""Latir!"" para o cachorro. O cachorro, ao receber a mensagem, executa o método latir(), que faz o cachorro emitir o som de um latido.
" O que é um método na POO?
Em Programação Orientada a Objetos (POO), um método é como uma função que pertence a uma classe. Ele define uma ação ou comportamento que os objetos daquela classe podem realizar.
Exemplo:
Imagine a classe Cachorro:
  • Atributos: nome, raca, idade, cor
  • Método: latir()
O método latir() define a ação que um objeto Cachorro pode realizar. Ao chamar o método latir() em um objeto Cachorro, ele irá emitir o som de um latido.
O que é um atributo na POO?
Em Programação Orientada a Objetos (POO), um atributo é uma propriedade ou característica de um objeto. É uma variável que armazena um valor específico para um determinado objeto, representando um dado ou informação sobre ele.
Exemplo:
Imagine a classe Cachorro:
  • Atributos: nome, raca, idade, cor

Polimorfismo
" "
Polimorfismo significa ""muitas formas"". Em POO, permite que um objeto se comporte de maneiras diferentes, dependendo do seu tipo real.
Exemplo:
Imagine uma função fazerSom():
  • Se o objeto for um Cachorro, a função faz o cachorro latir (Au! Au!).
  • Se o objeto for um Gato, a função faz o gato miar (Miau!).
Benefícios:
  • Flexibilidade: Permite que o código seja mais adaptável e reutilizável.
  • Manutenção: Facilita a adição de novos tipos de objetos sem afetar o código existente.
Exemplo Prático:
class Animal:
  def fazerSom(self):
    print(""Som genérico"")

class Cachorro(Animal):
  def fazerSom(self):
    print(""Au! Au!"")

class Gato(Animal):
  def fazerSom(self):
    print(""Miau!"")

def chamarAnimal(animal):
  animal.fazerSom()

cachorro = Cachorro()
gato = Gato()

chamarAnimal(cachorro) # Saída: Au! Au!
chamarAnimal(gato) # Saída: Miau!
Ponto-chave:
O código chamarAnimal() não precisa saber o tipo específico do animal, pois o polimorfismo garante que o método fazerSom() correto seja chamado com base no tipo real do objeto.

Tipos de Operadores em Programação

"

  • Subtração (-): Subtrai um valor numérico de outro. Exemplo: 10 - 3 = 7

  • Divisão (/): Divide um valor numérico por outro. Exemplo: 10 / 2 = 5
  • Módulo (%): Retorna o resto da divisão inteira de um valor por outro. Exemplo: 7 % 3 = 1

Conversão de Tipos: Conversão Implícita
" "
A conversão implícita é um processo automático realizado pela linguagem de programação, onde um tipo de dado é convertido para outro tipo de dado sem a necessidade de intervenção explícita do programador.
Exemplos:
1. Promoção em operações:
  • Quando dois operandos de tipos diferentes são combinados em uma operação, o tipo de dado de menor precisão é promovido para o tipo de dado de maior precisão.
Exemplo:
3 + 2.5 = 5.5
  • 3 é um inteiro (int) e 2.5 é um número de ponto flutuante (float).
  • O inteiro 3 é promovido para um float antes da operação de adição.
  • O resultado da operação é um float (5.5).
 Promoção em atribuições:
  • Quando um valor de um tipo de dado de menor precisão é atribuído a uma variável de um tipo de dado de maior precisão, o valor é promovido para o tipo da variável.
Exemplo:
float result = 6;






  
  • 6 é um inteiro (int).
  • A variável result é do tipo float.
  • O inteiro 6 é promovido para um float (equivalente a 6.0) antes de ser atribuído à variável result.

Relação de Composição

" "A composição é um tipo de relacionamento entre classes em programação orientada a objetos (POO) que representa uma relação de ""tem um"" forte. Uma classe ""possui"" outra classe, e a classe ""possuída"" depende da classe ""possuidora"" para sua existência.

Exemplo:
Imagine um sistema de gerenciamento de carros:
  • Carro: Representa um carro, com atributos como marca, modelo e motor.
  • Motor: Representa o motor do carro, com atributos como potência e tipo.
Relação:
Um Carro ""tem um"" Motor, e o Motor não pode existir independentemente do Carro.
Diagrama UML:
+-------------+
     | Carro       |
     +-------------+
         ^
         |
         |  composição
         |
     +-------------+
     | Motor       |
     +-------------+
" "
Quando se pode determinar que existe uma relação de Composição?
" "
Uma relação de Composição é caracterizada por:
  • Propriedade: Uma classe possui outra classe como um atributo, representando uma relação de ""tem um"".
  • Dependencia: A classe ""possuída"" depende da classe ""possuidora"" para sua existência. A classe ""possuída"" não pode existir independentemente da classe ""possuidora"".
  • Ciclo de vida: A classe ""possuída"" é criada e destruída junto com a classe ""possuidora"".

Relação:
Um Carro ""tem um"" Motor, e o Motor não pode existir independentemente do Carro.

Um Pedido ""usa"" a classe Cliente, pois o pedido precisa de um cliente para ser associado. No entanto, o Pedido não possui o Cliente.

Visibilidade

Visibilidade, no contexto da programação orientada a objetos (POO), refere-se ao nível de acesso aos membros (atributos e métodos) de uma classe. A visibilidade determina quais partes do código podem acessar e modificar esses membros.

Níveis de Visibilidade:
  • Público (public): Membros públicos são acessíveis por qualquer parte do código, incluindo classes externas.
  • Privado (private): Membros privados só são acessíveis por métodos da própria classe.
  • Protegido (protected): Membros protegidos são acessíveis por métodos da própria classe e de suas subclasses.
  • Pacote (package): (Algumas linguagens) Membros do pacote são acessíveis por classes dentro do mesmo pacote.

Temporalidade

"A temporalidade, no contexto da programação orientada a objetos (POO), se refere à duração da relação entre classes, ou seja, o tempo em que uma classe interage ou depende de outra classe.

Exemplos:
  • Classe Carro e Motor: A relação entre a classe Carro e a classe Motor é permanente, pois o carro ""possui"" um motor.
  • Classe Pedido e Cliente: A relação entre a classe Pedido e a classe Cliente pode ser transitória, pois o pedido pode ser criado e processado sem que o cliente esteja ativo no sistema.

Versatilidade

Versatilidade, no contexto da programação orientada a objetos (POO), refere-se à flexibilidade e capacidade de adaptação de uma classe ou módulo. Uma classe versátil pode ser utilizada em diferentes contextos e com diferentes tipos de objetos, sem a necessidade de grandes modificações.

Exemplo:
Imagine uma classe Calculadora que oferece operações matemáticas básicas. Uma classe Calculadora versátil:
  • Aceita diferentes tipos de dados: Pode realizar operações com inteiros, números de ponto flutuante e decimais.
  • Oferece diferentes operações: Pode realizar adição, subtração, multiplicação, divisão, etc.
  • Pode ser estendida: É possível adicionar novas operações, como raiz quadrada ou logaritmo, sem afetar o código existente.
